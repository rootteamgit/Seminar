# 企画書（kikakusho）作成ルール

セミナー企画書をExcel形式で作成する際のすべてのルールと構成。

---

## 📋 企画書の必須7フィールド

すべてのセルが埋まっていること。空欄・「要検討」で完成させない。未確定項目は 🔴 マークする。

| # | フィールド | 内容 | 入力形式 |
|----|-----------|------|---------|
| 1 | セミナータイトル | セミナーの正式名称 | テキスト |
| 2 | リード文 | 「〇〇で悩んでいませんか？」から始まる導入 | 2〜3文、参加者の悩みベース |
| 3 | ターゲット | 「このような方向け」の記述 | 箇条書き（職種・役職・悩み） |
| 4 | アジェンダ | セクション名 + 時間配分 | 「〇〇（XX分）」形式 |
| 5 | 解決できる課題 | セミナー参加で何が解決するか | 具体的成果（例：「3ステップで新人研修を30%削減」） |
| 6 | 登壇者・会社情報 | 登壇者の肩書・プロフィール + 会社情報 | 公開用テキスト |
| 7 | 集客目標・CTA・アーカイブ | 目標参加人数・主CTAボタンテキスト・後日配信有無 | 「100人 / CTA: 〇〇 / アーカイブ有」 |

---

## 🎯 テーマ決定フロー（STEP 1〜6）

企画書を書く前に必ずテーマを確定させること。以下6ステップを順番に通す。

### STEP 1：登壇者プロファイルを固める
- 氏名・役職・専門分野
- 実績・数字・ストーリー
- 「この人が語れるテーマ」を3〜5個列挙

### STEP 2：ターゲットを設定する
- 登壇者の年齢 ± 12歳が基本レンジ
- 第1〜第3ターゲットを「職種・役職・悩み」で定義
- **第1ターゲットに100%刺さることだけ考える**

### STEP 3：ブランド（約束）を言語化する
- 「このセミナーが参加者に何を約束するか」を1〜2文で記述
- 「〇〇ができるようになる」という具体性が必須

### STEP 4：テーマ候補を3分類で出す
- **恒久テーマ**: 業界ずっと需要がある定番
- **時事テーマ**: 今まさに話題のニュース・法改正
- **季節テーマ**: 毎年繰り返す時期的ニーズ

3分類から最低1個ずつ候補を出す。

### STEP 5：ブランドとの掛け算でテーマを1本に絞る

以下すべてYesなら採用:
- [ ] 登壇者が実体験・実績をもって語れるか？
- [ ] ブランドの「約束」を果たせるテーマか？
- [ ] 第1ターゲットの「今の悩み」に直接刺さるか？
- [ ] 「枠の外」のターゲットも意識した要素を入れているか？

### STEP 6：過去データと照合して確定する
- 過去の類似テーマ：申込数・満足度は？
- 競合・同業他社：同じテーマをやっているか？（需要の証明）
- SNS・検索：話題量・ボリュームはあるか？

---

## 🔴 エコーチェンバー防止チェックリスト

毎回テーマ選定時に確認：

- [ ] 過去3回と被っていないか？
- [ ] ターゲットが毎回同じ層になっていないか？
- [ ] SNSで話題量が十分か？
- [ ] 「いつもと違う視点」の要素は入っているか？

---

## 📊 高パフォーマンステーマの条件（実績ベース）

申込・CV数が最も高いテーマの特徴：

| 条件 | 詳細 |
|------|------|
| HR・研修 × AI活用 | 「新人研修をAIで効率化」系が最高CVR |
| コスト削減メッセージ | 「30%削減」等の具体数字が入るとCV上がる |
| ライブデモ | 実演を含むセッションはCV率 +30%以上 |
| 実例・ケーススタディ | 抽象的・理論系は低パフォーマンス |

## 💌 リード文の書き方

参加者が最初に目にするテキスト。ここでフック＆刺さらなければ離脱。

**NG例**:
- 「AIで業務効率化する方法」（誰に向けてるか不明）
- 「RootTeamの製品紹介」（内向き）

**OK例**:
- 「新人研修に毎年100時間かかっていませんか？Z世代のモチベーションを引き出しながら、AI活用で50%削減する3つのステップ」

**テンプレート**:
```
[ターゲット層へのダイレクトな悩み]＋[具体的成果]＋[差別化ポイント]

例：
新人研修を作る人事担当者、毎年100時間の教材作成に
困っていませんか？ ← 悩み

本セミナーでは、AI動画化ツール TalkSlide を使い、
スライドから30分で研修動画を作る方法を3ステップで紹介します。 ← 具体成果

参加企業の事例：平均50%の作成時間削減を実現。 ← 差別化

```

---

## 🎓 アジェンダ記述形式

時間配分を明示。合計時間が企画書タイトルの時間と一致していること。

```
【60分セミナー】
- オープニング・自己紹介（5分）
- 新人研修の課題　実例3社（10分）
- AIで研修動画を作るステップ　ライブデモ（20分）
- Q&A・アーカイブ案内（5分）
```

計: 40分（＋時間に余裕あり）

## 📌 共催パートナーがいる場合の企画書

| 項目 | 記述内容 |
|------|---------|
| パートナー名 | 正式社名 |
| パートナー担当者 | 氏名・連絡先 |
| CTA分岐 | 「RootTeamのCTA」「パートナーのCTA」を明記 |
| 申込フロー | どちらのLPから誘導するか |
| アーカイブ・資料配布 | 共有・個別配信どちらか |

---

## 🔴 未確定フラグの使い方

項目が未決定の場合のみ、セルに `🔴 要確認: [具体的に何が未決定か]` と書く。

**例**:
- `🔴 要確認: 落合さんに日時確認待ち（6/20まで）`
- `🔴 要確認: 森永さんの顔写真のURL`

企画書を提出する段階では🔴がゼロになっていること。

---

## 📁 ファイル命名規則

```
第N回セミナー_企画書_YYMMDDvX.xlsx

例：
第15回セミナー_企画書_260626v1.xlsx
第15回セミナー_企画書_260626v2.xlsx  ← 修正版は v2, v3...

共催の場合:
第15回セミナー_企画書_日刊スポーツ共催_260626v1.xlsx
```
