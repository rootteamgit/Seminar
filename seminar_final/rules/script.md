# 台本（Script）作成ルール

セミナー進行台本を .docx で作成する際のすべてのルール・記法・品質基準。

---

## 🎯 台本の目的

- **本番中の指針**: 登壇者が見て「今何を話すか」を即判定
- **進行管理**: 運営メンバーが時間・スライド・Q&Aを管理
- **アーカイブ字幕**: セミナー後のテロップ・誤字チェック用

---

## 📝 台本記法（統一記号）

| 記号 | 意味 | 使用例 |
|------|------|--------|
| 🟦 | 森永・澤居が話す | 🟦 森永：「本日はお集まりいただきありがとうございます」 |
| 🟩 | 羽生MC・司会が話す | 🟩 羽生：「では澤居さん、よろしくお願いします！」 |
| 🟧 | 上出キャンディが話す | 🟧 上出：「AIの活用方法について…」 |
| 📊 | スライド切替（番号・タイトル併記必須） | 📊 S3：TODAY'S THESIS |
| 📝 | 運営メモ（参加者には見えない） | 📝 ← Zoomチャットで「質問受付開始」通知 |
| 🔴 | 要確認・準備事項（未解決） | 🔴 デモPCセットアップ確認待ち |

### docxの登壇者色分け（TextRun color）
| 登壇者 | 色コード |
|--------|---------|
| 森永・澤居 | #1E3A8A（青） |
| 羽生MC・司会 | #065F46（緑） |
| 上出キャンディ | #B45309（アンバー） |

## 📄 台本の標準構成（30分セミナー・第10回スタイル）

### 必須要素
- **TODAY'S THESIS**: S3で今日の核心を1枚で言い切る
- **さくら質問**: 3問（回答案付き）
- **時間管理**: 各章の累計時間を明記
- **準備メモ**: チャットに貼るURL・実演リスト

### 標準時間配分（30分）
```
① イントロ：3分（00:00〜03:00）
② 本論前半：5分（03:00〜08:00）
③ 本論中盤：9〜11分（08:00〜17:00〜19:00）
④ 本論後半：7〜8分（19:00〜26:00）
⑤ CTA・Q&A：4〜5分（26:00〜30:00）
```

## 🎭 登壇スタイル別のルール

### 森永・澤居の単独登壇（羽生MC付き）
- 羽生が進行役として章の橋渡しをする
- 羽生のセリフは澤居/森永の発言を1〜2文で意味を反芻してから次に進む
- アジェンダのゴール宣言を羽生が行う（「今日のゴールは〜」）

### 上出キャンディとのリレー形式
- 掛け合いなし・独立して話すスタイル
- 羽生MCは章末の橋渡し一言のみ
- TalkSlide動画化は上出パートに入れず、森永パートへ
- 台本は【上出パート】【森永パート】を章見出しで明記
- 上出さんが修正した内容は絶対に変更しない

### 澤居×森永の対談形式（第10回・第13回スタイル）
- 羽生MCが司会進行
- 各章でさくら質問を使ってテンポを作る
- 実演デモは澤居が画面共有して行う

## 📄 台本の品質チェックリスト

提出前に必ず確認：

- [ ] 企画書のアジェンダと章構成が完全一致しているか？
- [ ] すべてのスライドが台本上に📊で明記されているか？
- [ ] スライド枚数と📊の数が一致しているか？
- [ ] 登壇者の色分けが正しいか（🟦森永/澤居、🟩羽生、🟧上出）？
- [ ] さくら質問が3問・回答案付きで入っているか？
- [ ] 時間管理（累計時間）が各章に明記されているか？
- [ ] 準備メモ（URL・実演リスト）が末尾に入っているか？
- [ ] 🔴要確認項目がゼロになっているか？
- [ ] TODAY'S THESISがS3に入っているか？

## 📐 スクリプト長さの目安

| セミナー時間 | 想定文字数 | 1分あたり |
|-------------|----------|---------| 
| 30分 | 2,500〜3,000字 | 85-100字/分 |
| 60分 | 5,000〜6,000字 | 85-100字/分 |

---

## 💬 話し言葉のルール

- 「〜します」より「〜するんです」「〜しましょう」
- 固い漢語より日常語を使う
- 1文が80文字以内に収まるようにする

---

## 📝 ファイル命名規則

```
第N回セミナー台本_vX.docx

例：
第18回セミナー台本_v1.docx
第18回セミナー台本_v2.docx  ← 修正版
```
