# スライド設計・デザインルール

PowerPoint（.pptx）でセミナースライドを制作する際のすべてのルール。pptxgenjsで生成する場合も同じシステムを適用。

---

## 🎨 デザインシステム（pptxgenjs標準）

### スライドサイズ
```
横:  10 inch
縦:  5.625 inch
アスペクト比: 16:9 （ワイドスクリーン）
```

### カラーパレット
| 用途 | カラー | 16進法 |
|------|--------|--------|
| 背景 | ネイビー | #0D1B2E |
| アクセント1 | シアン | #29C5F6 |
| アクセント2 | ピンク | #EC2D7B |
| テキスト（基本） | ホワイト | #FFFFFF |
| テキスト（サブ） | ライトグレー | #CCCCCC |

### フォント
- **メイン**: Yu Gothic（游ゴシック）
- **フォールバック**: Arial（Yu Gothic未対応環境用）
- **日本語**: Yu Gothic，游ゴシック Bold
- **英数**: Arial, Calibri

### マージン（定数）
```
MARGIN = 0.55 inch （すべてのスライドで一定）

スライド実使用域:
left:   0.55 in
right:  9.45 in （10 - 0.55）
top:    0.55 in
bottom: 5.075 in （5.625 - 0.55）
```

## 📐 スライドテンプレート

### パターン1: タイトルスライド

```
[背景: 背景グラデーション（ネイビー → 濃紺）]

タイトル:
  フォントサイズ: 54pt
  色: #FFFFFF
  位置: 中央上部（Y: 2.0 in）
  太さ: Bold

サブタイトル:
  フォントサイズ: 28pt
  色: #29C5F6 （シアン）
  位置: 中央（Y: 3.2 in）

[装飾: 右下に小さなアクセント図形]
```

### パターン2: コンテンツスライド

```
[背景: #0D1B2E ]

セクションタイトル:
  フォントサイズ: 32pt
  色: #FFFFFF
  位置: 左寄せ（X: 0.55 in）
  マーク: 左に色付きラインバー（幅0.08in, 色: #29C5F6）

本文:
  フォントサイズ: 18pt
  色: #FFFFFF
  行間: 1.5倍

箇条書き:
  • マーク色: #EC2D7B
  • インデント: 0.3 in
  • 1行最大70文字
```

### パターン3: 画像・デモスクリーンショット

```
[背景: #0D1B2E ]

キャプション:
  フォントサイズ: 16pt
  色: #29C5F6
  位置: 画像上部

画像/スクリーンショット:
  最大幅: 8.9 in
  中央寄せ
  
説明文:
  フォントサイズ: 14pt
  色: #CCCCCC
  位置: 画像下
```

## 🔑 スライド設計の基本原則

### 1分1スライドの「適度な密度」

| 情報量 | 目安 |
|--------|------|
| 少ない | テキスト3行以下＋画像 |
| 中程度 | テキスト5〜7行＋箇条書き3個 |
| 密 | 細かいテキスト＋複数要素（避ける） |

**原則**: 1スライド最大 `3セクション` か `5行の箇条書き` か `1メインビジュアル`

### 2. テキストサイズの階層

| 要素 | サイズ |
|------|--------|
| スライドタイトル | 32〜40pt |
| 見出し | 24〜28pt |
| 本文 | 18〜20pt |
| 補足・キャプション | 12〜16pt |

18pt以下は「読み込み」が必要になる → ライブセミナーでは避ける

### 3. 色使いのルール

- **背景**: 常に #0D1B2E （ネイビー）
- **強調**: シアン（#29C5F6）または ピンク（#EC2D7B）
- **多色使用**: 1スライドに3色以上のアクセント色は避ける
- **テキスト対比**: 背景ネイビーに対して白文字は必須（アクセス性）

### 4. 図表・チャート

- グラフ内の凡例: 12pt以上
- チャートの背景: 透明 または ネイビー
- 線幅: 2pt以上（スクリーンに映すときの見易さ）

## 📊 スライド構成の一般的なパターン

### セミナー全体の標準構成（60分）

```
1. タイトル（1スライド）
2. アジェンダ・本日のゴール（1スライド）
3. 背景説明（3〜4スライド）
4. メインテーマ解説（8〜10スライド）
   ├─ ライブデモ領域（4〜5スライド）
   └─ ケーススタディ（3スライド）
5. Q&A/まとめ（1スライド）

合計: 15〜18スライド
```

（実際のセミナー時間で 60分 ÷ スライド数 = 約3.5分/スライド）

## 🚨 NG パターン

| NG | 理由 | 修正案 |
|----|------|--------|
| 1スライドにテキスト10行以上 | 読む時間が必要 → 視聴者離脱 | 2スライドに分割 |
| 背景に複雑なパターン | テキストが読みづらい | ネイビー無地に統一 |
| 赤と青の同時使用 | 色覚多様性に対応不足 | シアンとピンクで代替 |
| スライドサイズが異なる | プロファイル作成時にエラー | 常に 10×5.625 in |
| フォント混在（Yu Gothic + MS明朝） | テーマ感が壊れる | Yu Gothic統一 |
| 画像圧縮なし | ファイルサイズ100MB超 | 画像は72dpi, JPG圧縮 |

## 💡 デモスクリーンショット・画像の品質基準

### 画像の仕様

| 項目 | 要件 |
|------|------|
| 解像度 | 1920×1080 以上（フルHD） |
| 圧縮 | JPG品質 85% 以上 |
| DPI | 72 DPI（Web・スクリーン表示用） |
| ファイルサイズ | 1枚500KB以下が目安 |

### スクリーンショット時の注意

- 不要なウィンドウは閉じる
- タスクバー・メニューバーは非表示
- 日本語テキストは14pt以上で表示
- 個人情報（メールアドレス・パスワード）は必ずマスク

## 🎬 ライブデモ用スライドの作り方

本番中に「スライド内で実演」する場合:

1. **スクリーンショット** + **矢印・注釈** の組み合わせ
   - UI上の重要部分に赤い矢印を追加
   - 「ここをクリック」というキャプションを付ける

2. **ステップバイステップのスライド分割**
   - 1操作 = 1スライド
   - 「Step 1 of 5」という表示で進捗を示す

3. **時間余裕を確保**
   - デモ用スライド前に「デモスライド（5分）」という表示
   - 本番中にスキップ可能な構成にする

## 📝 ファイル命名規則

```
第N回セミナー_資料_セミナータイトル_vX.pptx

例：
第15回セミナー_資料_Z世代向け新人研修_v1.pptx
第15回セミナー_資料_Z世代向け新人研修_v2.pptx  ← 修正版

共催の場合:
第15回セミナー_資料_日刊スポーツ共催_260626v1.pptx
```
