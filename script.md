# 台本（Script）作成ルール

セミナー進行台本を .docx で作成する際のすべてのルール・記法・品質基準。

---

## 🎯 台本の目的

- **本番中の指針**: 登壇者が見て「今何を話すか」を即判定
- **進行管理**: 運営メンバーが時間・スライド・Q&Aを管理
- **アーカイブ字幕**: セミナー後のテロップ・誤字チェック用

---

## 📝 台本記法（統一記号）

| 記号 | 意味 | 使用例 |
|------|------|--------|
| 🟩 | 森永（RootTeam COO）が話す | 🟩 「本日はお集まりいただきありがとうございます」 |
| 🟦 | 上出さん／外部登壇者が話す | 🟦 「AIの活用方法について…」 |
| 📊 | スライド切替（番号・タイトル併記必須） | 📊 **スライド 3「新人研修の課題」に切替** |
| 📝 | 運営メモ（参加者には見えない） | 📝 ← Zoomチャットで「質問受付開始」通知 |
| 🔴 | 要確認・準備事項（未解決） | 🔴 森永さんのデモPCセットアップ確認待ち |
| ⏱️ | タイミング・時間指示 | ⏱️ **ここまで 15分経過** |
| 🎬 | ライブデモ開始 | 🎬 **ここから TalkSlide デモ開始（5分予定）** |

## 📄 台本の標準構成（60分セミナー）

### セクション1: 開始〜5分

```
【セクション: オープニング】
⏱️ 00:00-05:00

🟩 「こんにちは、本日はお忙しい中お越しいただきありがとうございます。
RootTeam COOの森永です。本日のテーマは『Z世代向け新人研修の作り方』です。」

📊 **スライド 1「本日のテーマ」に切替**

🟩 「このセミナーに参加されると、次の3つが分かります：
　1. 新人研修で Z世代が離脱する理由
　2. AIを使った教材作成の3ステップ
　3. 実際に50%の時間削減を達成した事例」

📝 ← Zoom では「ウェビナー開始」ステータス確認
📝 ← チャット監視開始（質問は後ほど対応と告知）
```

### セクション2: 背景・課題提示（5〜20分）

```
【セクション: 新人研修の課題】
⏱️ 05:00-20:00

🟩 「まず、新人研修の現状をデータで見てみます。」

📊 **スライド 3「過去3年の新人研修時間」に切替**
[グラフ画像: 2023年120h → 2024年125h → 2025年130h]

🟩 「年々増えていますね。理由は3つです：」

📊 **スライド 4「3つの課題」に切替**

🟩 「1つ目：Z世代は長い説教を嫌う。平均離脱時間は7分です。
　2つ目：教材作成に時間がかかる。スライド作成だけで平均50時間。
　3つ目：更新が遅い。市場の変化に対応できていない。」

📝 ← この発言中、右下にタイマー表示（あと15分）
```

### セクション3: メイン解説 + デモ（20〜50分）

```
【セクション: AI活用の3ステップ】
⏱️ 20:00-50:00

🟩 「では、AIを使った解決方法を紹介します。3ステップです。」

📊 **スライド 6「ステップ1: スライド準備」に切替**

🟩 「ステップ1は『スライド準備』。
　新人研修の内容を PowerPoint で作ります。
　この時点では見た目は気にしなくてOK。」

📊 **スライド 7「ステップ1 実例」に切替**
[スクリーンショット: 簡素な PowerPoint スライド]

🟩 「こんな感じでいいんです。大事なのは『内容』です。」

🎬 **ここから TalkSlide デモ開始（デモ用スライド5〜8）**

🟩 「では実際にやってみます。
　このスライドを TalkSlide にアップロードして…」

📊 **スライド 9「デモ1: ファイルアップロード」に切替**
[スクリーンショット: TalkSlide の UI]

🟩 「こうして、設定を選んで『生成』をクリック。」

📊 **スライド 10「デモ2: 生成実行中」に切替**

📝 ← ここで10秒待機（実際の処理を見せる）

🟩 「30秒で完成。」

📊 **スライド 11「デモ3: 完成動画プレビュー」に切替**
[動画埋め込みまたはスクリーンショット]

🟩 「このように、スライドが自動的に喋る動画に変わります。
　スライドが多いほど、TalkSlide は自動で最適な長さに調整するんです。」

🎬 **デモ終了**
```

### セクション4: Q&A・まとめ（50〜60分）

```
【セクション: Q&A・まとめ】
⏱️ 50:00-60:00

🟩 「ご質問ありますか？チャットにお気軽に。」

📝 ← Zoom チャットから質問を読む（最大3問）
📝 ← 質問がなければそのまま進行

🟩 「それでは本日のまとめです。」

📊 **スライド 15「本日のまとめ」に切替**

🟩 「3つのステップで、新人研修動画を素早く作ることができます。
　実際にこの方法で、平均50%の時間削減を達成しています。」

📊 **スライド 16「CTA: 無料デモ予約」に切替**

🟩 「本日の内容に興味をお持ちでしたら、
　ぜひ無料デモをお試しください。
　予約リンクはチャットに貼っています。」

📝 ← Zoom チャットに予約リンクを貼付
📝 ← アーカイブ動画の公開日時を案内

🟩 「本日はご参加ありがとうございました！」

⏱️ 終了
```

## 🚨 台本の品質チェックリスト

提出前に必ず確認：

- [ ] すべてのスライドが台本上に 📊 で明記されているか？
- [ ] スライド番号の順序が連続しているか？（飛び番なし）
- [ ] 🟩 と 🟦 の登壇者が間違っていないか？
- [ ] 実演が入っている場合、🎬 でマーク→実演終了をマークしているか？
- [ ] ⏱️ タイミングの合計が予定時間（±5分以内）になっているか？
- [ ] 🔴 要確認項目がゼロになっているか？
- [ ] 1文が80文字以上になっていないか？
- [ ] 敬語・カジュアル表現がセミナー対象に合っているか？

## 📐 スクリプト長さの目安

セミナーの実際の時間と台本の文字数の相関関係:

| セミナー時間 | 想定文字数 | 1分あたり |
|-------------|----------|---------|
| 30分 | 2,500〜3,000字 | 85-100字/分 |
| 60分 | 5,000〜6,000字 | 85-100字/分 |
| 90分 | 7,500〜9,000字 | 85-100字/分 |

（話す速度により変動。ゆっくり：60字/分、通常：90字/分、早い：110字/分）

## 💬 話し言葉の作り方

台本は「読まれる」ものなので、自然な話し言葉で書くこと。

**NG: 硬い書き言葉**
```
「新人研修に関しては、莫大な時間的コストが発生している。」
```

**OK: 自然な話し言葉**
```
「新人研修って、毎年すごい時間がかかってますよね。」
```

**ルール**:
- 「〜します」より「〜するんです」「〜しましょう」
- 「一方」より「一方で」
- 「について」より「について」→ OK（中立）
- 固い漢語より日常語を使う

## 🎬 デモ・ライブ実演の記述

デモが複数ステップある場合、スライド1枚=操作1ステップ で分割すること。

```
🎬 **デモ開始: TalkSlide ファイル生成**

📊 **デモスライド 1「Step 1: ファイルを選択」**
🟩 「まず、このファイルを選択して…」

📊 **デモスライド 2「Step 2: 設定を入力」**
🟩 「こうして設定項目を埋めます。」

📊 **デモスライド 3「Step 3: 生成実行」**
🟩 「そして『生成』をクリック。」

📊 **デモスライド 4「Step 4: 完成確認」**
🟩 「こんな感じで完成します。」

🎬 **デモ終了**
```

本番中に「スキップ可能」な構成にすること（デモに時間がかかった場合）。

## 📝 ファイル命名規則

```
第N回セミナー_台本_セミナータイトル_vX.docx

例：
第15回セミナー_台本_Z世代向け新人研修_v1.docx
第15回セミナー_台本_Z世代向け新人研修_v2.docx  ← 修正版

共催の場合:
第15回セミナー_台本_日刊スポーツ共催_260626v1.docx
```

## 🔄 台本の修正ルール

### 修正版 v2, v3... の作り方

1. 登壇者からフィードバック受ける
2. 修正内容を **修正内容の記録** に記入
3. ファイル名を `_v2.docx` にして再出力
4. 修正前後を赤字で表示して手元保管（バージョン管理用）

**修正内容の記録** テンプレート:
```
【修正版 v2】
- 🟩 森永の冒頭文が長すぎるとの指摘 → 短縮
- デモのステップが7個 → 5個に削減
- Q&A時間の調整: 3分 → 5分に延長
```
