## 🏗️ 第12回デザインの構造（必ず踏襲）

```
スライド全体に以下の要素を配置:

1. 左ピンク縦バー
   x=0, y=0, w=0.18*SX, h=全高, fill=#D9369E

2. 上部ブルーライン
   x=0.18*SX, y=0, w=9.82*SX, h=0.12*SY, fill=#3A5BD9

3. 下部ブルーライン
   x=0, y=5.45*SY, w=10*SX, h=0.17*SY, fill=#3A5BD9

4. セクション番号バッジ（左上）
   x=0.45*SX, y=0.25*SY, w=0.55*SX, h=0.55*SY, fill=#D9369E

5. セクションタイトル（バッジ右）
   x=1.15*SX, y=0.27*SY, fontSize=22, bold, color=#FFFFFF

6. カード（コンテンツ用）
   fill=#1A2272 + 左ボーダー8px（シアン/ピンク/ブルーで使い分け）

7. 番号バッジ（各カード左上）
   01/02/03 形式、カード左ボーダーと同色

8. カード本文背景
   fill=#E8EEFF（薄い水色）でテキストエリアを囲む

9. フッター強調テキスト（下部）
   背景=#151A6A, テキスト=#F040B0, fontSize=12, bold
```

## 📐 スライドテンプレート

### パターン1: タイトルスライド
- 背景: #0D1147
- セクション番号バッジ（第N回）
- タイトル: 32〜42pt, bold, white
- サブタイトル: 18pt, cyan
- 区切りライン: fill=#3A5BD9
- 日時・登壇者: 12pt, #A0B4CC

### パターン2: コンテンツスライド（カード3枚）
- フレーム（左縦バー・上下ライン）
- セクションバッジ + タイトル
- カード3枚（各1.18*SY高さ）
- y位置: 1.1*SY / 2.48*SY / 3.86*SY
- フッターテキスト

### パターン3: 3列カード
- フレーム + セクションバッジ
- 横3列カード（各colW = 2.85*SX）
- フッターテキスト

### パターン4: 2列カード（比較・対比）
- フレーム + セクションバッジ
- 左右2枚のカード（各4.4*SX幅）

### パターン5: ステップカード（縦3枚）
- フレーム + セクションバッジ + サブ見出し
- STEP 1/2/3 形式のカード（各0.95*SY高さ）
- フッターテキスト

## 🔑 スライド設計の基本原則

### 1スライドの情報密度
- 1スライド最大3カードまたは5行の箇条書きまたは1メインビジュアル
- フッターには章全体のキーメッセージを1行で入れる

### テキストサイズの階層
| 要素 | サイズ |
|------|--------|
| スライドタイトル | 32〜42pt |
| セクション見出し | 22pt |
| カードヘッダー | 17〜18pt |
| 本文 | 13〜15pt |
| フッター | 12pt |

### スライドと台本の照合（必須）
- 台本の📊スライド切替マークと枚数が完全一致していること
- 作成後に必ず枚数をダブルチェックする

## 🚨 NGパターン

| NG | 理由 | 修正案 |
|----|------|--------|
| カード左ボーダーなし | 第12回テイストが崩れる | 必ず8pxボーダーを入れる |
| フッターなし | 章の締めメッセージが消える | 必ず下部にフォロー強調テキスト |
| フォントをYu GothicやAptos指定 | QA環境で崩れる | Calibri固定 |
| 背景色を#0D1B2Eにする | 旧仕様。基準は#0D1147 | 必ず#0D1147を使う |
| 1スライドにテキスト10行以上 | 視認性低下 | 2スライドに分割 |

## スライド生成の分業ルール（2026/06/26 確定）

### 基本方針
スライドの生成はNotebookLMに担わせる。Claudeはプロンプト作成とチェックのみを行う。

### 役割分担
Claude：企画書読み込み・台本生成・NotebookLM用プロンプト作成・check_agentでの乖離チェック・台本とスライドの対応確認
NotebookLM：スライド全枚数の生成（図解・挿絵・デザイン込み）
emaさん：NotebookLMでスライド生成・PDFをClaudeに渡す・GitHubへのpush

### 理由
- NotebookLMの方がスライドのビジュアルクオリティが高い
- Claudeの画像表示上限問題を回避できる
- 挿絵・図解・アイコンはNotebookLMに任せた方が整合性が取れる

### チェック手順
1. NotebookLMからPDFが上がってきたら枚数を確認する
2. 台本の📊スライド番号と対応しているか確認する
3. check_agentで企画書・台本・スライドの三者照合を実施する

## 🐍 python-pptxでの直接生成（NotebookLM代替フロー）

### いつ使うか
- NotebookLMで作成したPPTXはすべて**画像スライド**になるため、テキスト抽出・ミスチェック・プログラム編集が不可能
- 台本や企画書との照合チェックをClaudeで自動化したい場合はpython-pptxで直接生成する

### カラー定数（python-pptx版）
NotebookLM生成スライドとは別系統の「サイバー系」配色を使用。

```python
from pptx.dml.color import RGBColor

DARK_NAVY  = RGBColor(0x0D, 0x11, 0x47)  # 背景
DARK_PANEL = RGBColor(0x0F, 0x15, 0x3A)  # カード背景
NAVY2      = RGBColor(0x05, 0x09, 0x25)  # ヘッダー背景
CYAN       = RGBColor(0x00, 0xE5, 0xFF)  # アクセント1（メイン）
PINK       = RGBColor(0xFF, 0x2D, 0x9A)  # アクセント2
ORANGE     = RGBColor(0xFF, 0x6B, 0x00)  # アクセント3
WHITE      = RGBColor(0xFF, 0xFF, 0xFF)
GRAY       = RGBColor(0xAA, 0xBB, 0xCC)  # サブテキスト
```

### スライドサイズ
```python
from pptx.util import Inches
prs.slide_width  = Inches(13.33)
prs.slide_height = Inches(7.5)
```

### 共通部品パターン
```python
# ヘッダー（全スライド共通）
def hdr(slide, tag, title, tag_color=CYAN):
    # 上部カラーバー + タグ + タイトル + アクセントライン
    bx(slide, 0, 0, W, Inches(0.06), tag_color)
    bx(slide, 0, Inches(0.8), W, Inches(0.7), NAVY2)
    tx(slide, tag,   Inches(0.5), Inches(0.85), ..., sz=12, bold=True, col=tag_color)
    tx(slide, title, Inches(0.5), Inches(1.1),  ..., sz=24, bold=True)
    bx(slide, Inches(0.5), Inches(1.72), Inches(4), Inches(0.04), tag_color)

# カード（左アクセントバー付き）
def card(slide, x, y, w, h, accent_color=CYAN):
    bx(slide, x, y, w, h, DARK_PANEL)
    bx(slide, x, y, Inches(0.07), h, accent_color)
```

### 注意事項
- `python-pptx`で生成したPPTXはテキストレイヤーが生きているためミスチェック可能
- NotebookLMにインポートして再編集することも可能（デザイン上書きに注意）
- S14（CTA）のCTA文言修正など、後から編集が必要なスライドはpython-pptxで作ること
