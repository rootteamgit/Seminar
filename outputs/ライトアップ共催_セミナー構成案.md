# セミナー構成案：エージェンティック時代の到来と「AI社員」

> テーマ：1入力1出力のチャットAI時代から、自律的に仕事をこなすエージェント時代へ。
> その先に、職種特化エージェントが組織で人間のように働く未来。

---

## スライド01｜タイトル / つかみ
**見出し：「あなたの会社に "AI社員" が入る日」**

- 問いかけで開始：「もしAIが、指示を1回出すだけで、調べて・考えて・成果物まで持ってくる "部下" だったら？」
- 「それはもう未来の話ではなく、この1年で実際に起きたことです」と宣言。
- 本日の約束：①この1年で何が変わったのか（事実）②これから組織はどうなるのか（未来像）③私たちが実際に作っている "AI社員" を3体お見せします（具体）。

---

## スライド02｜時代の整理：この1年半の地殻変動
**見出し：「チャットAI」から「エージェント」へ**

タイムライン（横軸）で見せる：

| 時期 | 出来事 | 意味 |
|---|---|---|
| 〜2025春 | ChatGPT / Claude などチャットAI | **1入力1出力**の時代 |
| 2025/2月 | Claude Code（コーディングエージェント） | 自律的に連続作業する初の普及例 |
| 2025/4-5月 | OpenAI Codex | コーディング領域でエージェントが定着 |
| 2026/2月 | OpenClaw が大衆化（GitHub 10万★超） | **コーディングの枠を超えた汎用自律エージェント** |
| 2026/5月 | Google Gemini Spark | 24時間自律で動くパーソナルエージェント |

- ナレーション：「コーディングという狭い領域で実証され、OpenClawで一気に汎用へ広がった。今やAIは "答える" ものから "仕事をやり切る" ものに変わった」

---

## スライド03｜何が本質的に変わったのか（対比）
**見出し：「答えるAI」と「仕事をするAI」**

左右の対比図：

- **チャットAI時代**：指示 → 出力 → また指示 → 出力 …
  - 人間が毎回オーケストレーター。AIは "賢い検索 / 賢い下書き"。介在し続けないと止まる。
- **エージェント時代**：指示 → （自律的にプロセスを遂行：調査・判断・ツール操作・修正）→ 完成物
  - 人間は "レビュアー / 発注者"。AIが業務プロセスそのものを回す。

- ここで **実画面デモ or 短い動画**（Claude Code / Gemini Spark / 自社プロダクト）を1つ挟むと体感が段違い。

---

## スライド04｜なぜ "今" なのか（Why now）
**見出し：「なぜこの1年で急に起きたのか」**

- モデルが2つの能力を獲得したから：
  1. **長期タスクの遂行**（途中で目標を見失わず、何十ステップも自走する）
  2. **ツール使用**（ファイル・API・ブラウザ・社内システムを自分で操作する）
- これにより「賢いおしゃべり」から「実務の実行者」へ。バズワードではなく必然。

---

## スライド05｜未来像：組織の中でエージェントが "人間のように" 働く
**見出し：「AIは "ツール" から "同僚" になる」**

- これからの仕事体験：エージェントが一人ひとり、定型ルーティンを持ち、自律的に業務を回す。
- 人間の役割：指示する・レビューする・意思決定する（マネジメント側へ）。
- 組織図に "AIの席" が並ぶイメージ図。

---

## スライド06｜私たちの答え：職種特化エージェント
**見出し：「サイバー社員 / 無茶ぶりAI ── 職種ごとの "AI社員" を作る」**

- 汎用エージェントではなく、**職種に特化**して即戦力化するアプローチを取っている、と宣言。
- ここから3体を、バラバラにではなく **1本の業務フロー** として見せる（次スライド）。

---

## スライド07｜3体のエージェントを "業務フロー" で連結
**見出し：「エージェントだけで、会社の業務が一周する」**

リード獲得 → 育成・商談化 → プロダクト開発、の流れで配置：

1. **サイバーマーケター（マーケターエージェント）**
   - リード獲得・集客を自律実行。〔具体タスク例を1-2行〕
2. **ナーチャリングエージェント**
   - 獲得リードの育成・商談化を自律実行。〔具体タスク例〕
3. **開発エージェント**
   - プロダクト・機能開発を自律実行。〔具体タスク例〕

- メッセージ：「前半で語った "組織で働くエージェント" を、私たちはもう実装している」

---

## スライド08｜それらを束ねる基盤：グラントリー
**見出し：「複数のAI社員を、安全に・連携させて動かす」**

- 各エージェントに権限を渡し、外部SaaS（CRM / 広告 / メール / 開発基盤など）へ安全にアクセスさせる連携・権限管理基盤として紹介。
- 「1体ずつではなく、チームとして動かすための土台」と位置づけ。

---

## スライド09｜ビフォー / アフター
**見出し：「人がやっていた仕事が、こう変わる」**

- 対象業務 × 〔従来の工数・人数〕→〔エージェント運用後〕を数字で1枚。
- できれば顧客事例 or 自社実績を1つ。

---

## スライド10｜CTA（次の一歩）
**見出し：「あなたの会社の "最初のAI社員" を、一緒に」**

- 無料相談 / PoC / デモ依頼への導線を明示。
